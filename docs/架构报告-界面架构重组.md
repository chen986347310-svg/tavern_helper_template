# 架构报告: 雌堕合欢宗 界面架构重组

> 自动生成于实施完成后, 供后续维护和排障参考

## 当前权威架构结论（2026-05-22）

- **前端不是纯只读层**：前端可通过 `store.data` 执行已确认的即时交互，例如商城扣费、背包装备、库存变化和 `系统.待处理交互` 入队。
- **前端也不是剧情生成层**：前端不得调用 `generate()`，不得伪造心音、反噬扣数或剧情解锁；这些必须由用户下一条消息触发 AI 后，通过标准 MVU 变量块回流。
- **闭环主链路**：UI 点击 → 写入 `系统.待处理交互` → 用户发送下一楼层 → AI 读取队列并叙事承接 → 输出 `<UpdateVariable><JSONPatch>` → MVU 应用 → 前端显影。
- **同步边界**：`dist/雌堕合欢宗/index.html` 只代表状态栏 UI；世界书/角色卡是否生效取决于 SillyTavern 实际导入版本。当前 `tavern_sync.mjs` 缺失，必须显式记录替代同步方式。
- **场景枚举**：UI 与 MVU 字段只接受 `莲灯前苑`、`醉玉小筑`、`绮梦幽阁`；正文地点可更细，但必须映射回三大 UI 场景。

## 三、数据流架构

### 3.1 基础响应式链路

    MVU 变量 -> defineMvuDataStore(Schema) -> Pinia store -> 组件响应式读取
    组件已确认即时交互 -> store.data 赋值 -> watchEffect -> replaceVariables(klona(data)) -> 当前楼层 MVU 同步
    调试面板 -> store.data 修改 -> 即时反映到所有组件

- Schema 定义: schema.ts, Zod 4 幂等解析
- Store: store.ts 使用 defineMvuDataStore
- Guards: guards.ts 包含前端可判定的游戏逻辑函数

### 3.2 v4 UI → AI → MVU 闭环链路

    用户点击 UI
      ↓
    前端写入 系统.待处理交互（不调用 generate）
      ↓
    用户发送下一条消息
      ↓
    AI 读取待处理交互并叙事承接
      ↓
    AI 输出 <UpdateVariable><JSONPatch> 标准块
      ↓
    MVU 应用变量更新到最新楼层 stat_data
      ↓
    前端读取最新 MVU 并显影心音/剧情结果

- 前端可做即时表现：按钮颤动、心防反震提示、库存归零消散、装备显隐。
- 前端不可做剧情定论：`soul_whisper.text`、反噬扣数、剧情解锁、NPC 心境变化必须等 AI Patch。
- AI 必须清空 `系统.待处理交互`，否则下一楼层会重复执行同一交互。
- 裸 JSON Patch、YAML 摘要或“变量更新情况”只会留在正文，不属于可靠 MVU 数据通道。

- 图片加载完成后 opacity 淡入

## 十三、日/夜主题切换系统

### 架构概览

``
JS 层: useTheme() composable -> document.documentElement.setAttribute('data-theme', t)
CSS 层: :root { 暗色变量 } [data-theme=light] { 亮色变量 }
覆写层: html[data-theme=light] { 覆写 mixin 硬编码色 }
持久化: localStorage.getItem/setItem('theme')
```

### CSS 自定义属性体系

| 变量 | 暗色模式 | 亮色模式 | 用途 |
|------|---------|---------|------|
| --theme-bg-primary | #1a1008 | #f5f0e8 | 页面主背景 |
| --theme-bg-secondary | #2a1f14 | #ebe4d6 | 卡片/面板背景 |
| --theme-text-primary | rgba(232,196,74,0.85) | #2c1a08 | 主要文字 |
| --theme-text-secondary | rgba(180,150,100,0.5) | #5a3d1a | 次要文字 |
| --theme-gold | #e8c44a | #4a2d0a | 强调/数值 |
| --theme-border | rgba(212,160,23,0.2) | rgba(140,100,30,0.35) | 边框 |
| --npc-accent | var(--npc-color) | var(--npc-color-dark) | NPC 专属色 |

### mixin 硬编码色覆写

SCSS mixin（gold-book-bg、gold-foil 等）编译后产生硬编码颜色，CSS 变量无法覆盖。
_global.scss 中的 html[data-theme=light] 块使用 !important 和直接覆写解决：

- .scroll-frame / .page-view: 背景渐变覆写
- .gold-foil 衍生元素: 边框/阴影覆写
- 金色角落装饰/装饰线: 颜色覆写
- 滚动条: 轨道/滑块颜色覆写

### useTheme composable

```typescript
// src/雌堕合欢宗/界面/composables/useTheme.ts
const theme = ref<Theme>(getStoredTheme());
applyTheme(theme.value); // 模块加载时立即应用
function toggleTheme() { theme.value = ...; applyTheme(theme.value); }
```

### 主题切换过渡动画

`applyTheme()` 在切换前添加 `theme-transitioning` class 到 `document.documentElement`，触发全局 CSS transition（0.35s ease），350ms 后自动移除。防止快速切换时闪烁。

```css
.theme-transitioning,
.theme-transitioning * {
  transition: background-color 0.35s ease, color 0.35s ease, border-color 0.35s ease, box-shadow 0.35s ease !important;
}
```

### 切换按钮

SystemBar 右侧 40px 圆形按钮，SVG 太阳/月亮图标，使用 var(--theme-*) 自适应配色。

## 十五、v4 UI → AI → MVU 闭环架构补充（2026-05-22）

### 闭环分层

```text
玩家点击 UI
  ↓
前端写入 系统.待处理交互（不 generate）
  ↓
用户发送下一条消息
  ↓
AI 读取待处理交互并叙事承接
  ↓
AI 输出 <UpdateVariable><JSONPatch> 标准块
  ↓
MVU 应用变量更新到最新楼层 stat_data
  ↓
defineMvuDataStore 轮询并让前端显影
```

### 架构边界

- 前端可以修改当前楼层 `store.data`，用于购买扣费、背包装备、待处理交互入队等即时交互。
- 前端不能替 AI 写剧情结果；`soul_whisper.text`、剧情解锁、反噬扣数等必须来自下一楼层 Patch。
- AI Patch 必须完整包裹在 `<UpdateVariable>` 与 `<JSONPatch>` 中；裸 Patch 不属于可靠数据通道。
- `dist/雌堕合欢宗/index.html` 是状态栏 UI，不承载世界书规则；世界书同步失败时，UI 仍可正常渲染但 AI 不会按新规则输出。

### 实机排障顺序

1. 查最新楼层 iframe，如 `TH-message--6--0`。
2. 查正文尾部是否含 `<UpdateVariable>` / `<JSONPatch>`。
3. 查 `Mvu.getMvuData({ type: 'message', message_id: 'latest' })` 的实际变量。
4. 若正文有裸 Patch 但变量未回流，优先排查世界书/角色卡同步，而不是改 UI。
5. 若 AI 写入 schema 外枚举，优先加强变量规则与 schema 护栏。
## 十六、架构验收红线（2026-05-22）

- 如果 UI 已入队但下一楼层没有 `<UpdateVariable><JSONPatch>`，优先排查世界书/角色卡同步，不要误判为 Vue 组件缺陷。
- 如果最新 MVU 没有 `soul_whisper`，心音区域保持不显示是正确架构行为。
- 如果 AI 写入 `外门广场` 等非枚举场景，必须在世界书/输出规则层映射回三大 UI 场景，而不是放宽 schema。
- 如果修改 `schema.ts`，必须同步 `schema.json`、`initvar.yaml`、测试与世界书变量规则。
