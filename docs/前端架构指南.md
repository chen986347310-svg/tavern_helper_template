## 添加新功能指南

### 新增 NPC

需要同步修改 3 处:
1. _variables.scss: $npc专属色 map 加新颜色
2. schema.ts: z.enum 中加新名字
3. guards.ts: 攻略链数组加新名字

可选: assets/avatars/ 加新图片, 世界书/加新 NPC 设定

### 新增页面

需要联动 3 处:
1. App.vue: v-if 链加新条件
2. PageNav.vue: tabs 数组加新 tab
3. useNavigation: 无需改动 (已支持任意 tab 名)

### 新增数据字段

1. schema.ts: z.object 中加新字段
2. store 自动响应 (defineMvuDataStore 基于 Schema)
3. 组件直接通过 useDataStore().data.xxx 读取

### 新增 SCSS mixin

1. _mixins.scss: 定义新 mixin
2. 组件中 @use 'mixins' as * 后 @include 新 mixin

## 图片资源规范

| 类型 | 路径 | 规格 |
|------|------|------|
| 正道版 | assets/avatars/{npc名}.png | 512x512, upper body, face in upper third |
| 仙奴版 | assets/avatars/{npc名}_fallen.png | 同上 |
| 显示 | object-fit: cover + object-position: center 20% | |
| 容器 | 响应式宽度(百分比), 跟随 iframe 容器 | |
| 回退 | @error 时显示首字占位符 | |
  | 路径 | 相对路径(无开头/), 适配iframe子目录 | |

  ### Webpack 图片构建

  webpack.config.ts 中为 avatars 目录配置了专用 asset/inline 规则:

  ```
  include: path.resolve(import.meta.dirname, 'src/雌堕合欢宗/界面/assets/avatars')
  type: 'asset/inline'
  ```

  - 图片以 base64 data URI 内联进 JS bundle, 最终由 HtmlInlineScriptWebpackPlugin 内联进单个 HTML
  - 无外部文件输出, 酒馆 iframe 可直接渲染
  - 产物大小约 6 MB (10 张 512x512 压缩 PNG 的 base64, 原图已从 1254x1254 压缩 83%)
## 关键设计决策

| 决策 | 结论 |
|------|------|
| 导航 | ref + v-if, 不引入 vue-router |
| 数据 | Pinia store + MVU 双向同步 |
| 展开动画 | flex-wrap + width 过渡, 0.35s ease-in-out |
| 收起动画 | 0.55s ease-out (比展开慢 60%), 纯 opacity 动画无跳变 |
| 头像渐融 | ::before 伪元素叠加层, opacity 平滑过渡 |
| 仙奴切换 | 数据驱动, 无过渡动画 |
| 测试 | @vue/test-utils + happy-dom + vi.mock |
| 调试 | 单例 composable, Ctrl+Shift+D, 所有模式可用 |
