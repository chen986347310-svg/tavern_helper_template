# 服装商城重构映射表 v2：香艳暗示版

> 日期：2026-05-24  
> 范围：商城服装显示名、楼层分组、价格曲线、详情文案方向、AI 读取边界。  
> 状态：设计草案，基于 `服装商城重构映射表v1.md` 升级命名强度。

---

## 1. v2 调整目标

v1 命名偏雅致、偏安全，容易像“古风换皮服装店”。v2 的目标是让服装更像合欢宗内部流通的禁衣、法衣、祭衣：

1. **更强暗示**：名称里加入 `欲 / 涎 / 贴 / 锁 / 烙 / 献 / 契 / 牝` 等语义，让玩家一眼知道衣装的危险性。
2. **仍保留古风包装**：避免现代成人用品 SKU 命名，保持“宗门秘宝 / 法衣 / 禁制衣”的叙事感。
3. **不改逻辑 key**：`名称` 仍用于库存、装备、旧存档、门槛判断；`显示名` 才用于 UI。
4. **按楼层递进**：越高好感，名称越主动、越仪式化、越带身份归属意味。
5. **AI 分级承接**：商品文案可以成人向，但 AI 对具体角色描写必须结合角色设定、阶段、年龄与当前剧情，不机械套露骨描写。

### 牝奴服硬边界

- 只在 `系统.阶段 = 牝奴期` 时进入商城列表。
- 只能装备给 `玩家`，不能装备给任何 NPC。
- 不走 NPC 好感度门槛。
- AI 只在读取 `道具.装备.玩家` 时承接牝奴服，不得把牝奴服误写给 NPC。

## 5. 详情文案强度模板

详情页建议拆成三段，避免一口气写成直白说明：

### 5.1 衣装记述

用于玩家阅读，强调质感与场景。

示例：`湿雾贴身裙`

> 雾绡遇热便贴，行走时像一层湿雾缠住肌肤。灯影越暗，轮廓越清。

### 5.2 命理影响

用于说明装备后的叙事效果。

> 装备后，AI 可在角色入场、转身、紧张、被注视时参考衣料贴身与裙雾流动，不强制每回合复述。

### 5.3 禁制边界

用于说明门槛与适用对象。

> 需灵犀 50+；仅 NPC 可用；牝奴期玩家服另行显示。

## 6. AI 承接分级规则

### 6.1 成年/高阶 NPC

- 可承接衣料贴身、铃声、束缚、符纹发光、旁人目光、动作拘谨等成人向暗示。
- 不要求每回合都描写服装，但当角色入场、被注视、运动、情绪波动、施法时应自然触发。

### 6.2 低龄或不适配角色

- 商品可以存在，但 AI 对该角色承接时必须降级为含蓄描写。
- 只写衣料、灵纹、尴尬、压迫、羞怯、动作变化，不写露骨身体细节。
- 若世界书明确角色未成年或不适合 NSFW 承接，则服装效果应转为“宗门禁制/身份压迫/法衣异样”的非露骨表达。

### 6.3 玩家牝奴期

- 牝奴服只承接玩家自身状态变化。
- AI 应读取 `道具.装备.玩家` 与 `系统.阶段`，将衣装作为牝奴期身份、行动限制、仪式推进的视觉锚点。

## 7. UI 展示建议

### 商城列表

- 主标题显示 `v2 显示名`。
- 副标显示楼层：如 `诱形层 · 灵犀 50+`。
- 价格按楼层递增，命契层与牝奴层使用更强发光。
- 未达门槛商品可半透明展示，但按钮显示 `灵犀未至`。

### 背包列表

- 显示 `v2 显示名`。
- hover 或详情显示 `逻辑名` 只在调试模式可见。
- 装备/卸下仍使用逻辑名写入 MVU。

### 角色卡装备栏

- 显示 `v2 显示名`，避免角色卡出现后台 SKU。
- 如果装备多件，只显示 1-2 件核心衣装，其余折叠为 `另有禁衣若干`，防止角色卡拥挤。

---

## 8. 实现边界

- 不修改原 `名称`。
- 不修改购买、装备、卸下的核心生命周期。
- 不引入换装立绘。
- 不让牝奴服进入 NPC 装备目标。
- 先实现数据与显示闭环，再同步世界书 AI 读取规则。
