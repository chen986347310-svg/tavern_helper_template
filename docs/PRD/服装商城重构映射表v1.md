# 服装商城重构映射表 v1

> 日期：2026-05-24  
> 范围：商城服装命名、楼层分组、价格曲线、玩家牝奴服边界、AI 可读装备提示。  
> 状态：设计草案，待确认后进入实现计划。

---

## 1. 目标

当前服装商品名称偏直白，容易显得像后台 SKU，缺少从保守到暴露、从普通衣物到仪式法衣的攻略递进。本次重构目标是：

1. 保留现有 `名称` 作为逻辑 key，保证库存、装备、门槛、旧存档与测试不被破坏。
2. 新增玩家可见 `显示名`，让商城、背包、角色卡装备栏更贴合合欢宗世界观。
3. 按好感度门槛重排为楼层结构，让玩家感知“关系推进 → 可购买服装升级”。
4. 明确 `牝奴服` 为玩家专属，只在玩家进入 `牝奴期` 后显示与装备。
5. 为 AI 提供短句式 `AI提示`，使角色离场再入场时仍能读取穿着并自然承接描写。

## 3. 数据字段建议

| 字段 | 类型 | 是否新增 | 用途 |
|---|---|---:|---|
| `名称` | string | 否 | 稳定逻辑 key，不直接改名 |
| `显示名` | string | 是 | 商城、背包、角色卡玩家可见名 |
| `楼层` | string | 是 | 商城楼层分组与排序 |
| `适用对象` | `'NPC' \| '玩家' \| '全部'` | 是 | 约束玩家牝奴服与 NPC 服装边界 |
| `AI提示` | string | 是 | 世界书/变量规则读取后的叙事提示 |
| `好感度门槛` | number | 否/可调整 | 解锁门槛，普通 NPC 服装继续使用 |
| `价格` | number | 否/可调整 | 按楼层递增，形成奖励曲线 |

### 牝奴服限制

- 商城：仅 `系统.阶段 = 牝奴期` 显示。
- 背包：装备目标只显示 `玩家`。
- NPC：白芷、苏芸、纪兰、沈月秋、柳素衣不能装备牝奴服。
- AI：读取 `道具.装备.玩家` 时承接玩家牝奴期穿着；不得误写到 NPC 身上。

## 6. 商城 UI 分组建议

服装分类页不再只按 `系列` 平铺，而采用楼层分隔：

1. 顶部显示当前可见楼层玉签：`凡衣 / 微露 / 诱形 / 缚心 / 命契`。
2. 每个楼层有一条水墨分隔：显示楼层名、门槛语义、价格区间。
3. 未达门槛楼层可显示淡化预览，具体详情按钮提示 `灵犀未至`。
4. 牝奴期时追加 `牝奴` 楼层，视觉上与普通 NPC 服装分开。
5. 价格显示继续用灵石衬线数字，但详情页补充 `适用对象` 与 `AI提示`。

## 7. 服装详情页建议

详情弹窗应从“描述 + 价格”升级为：

- `显示名`：主要标题。
- `旧名/逻辑名`：不对玩家展示，调试面板可见。
- `楼层`：如 `诱形层`。
- `适用对象`：如 `NPC可用`、`玩家专属`。
- `门槛`：如 `灵犀 50+` 或 `牝奴期显化`。
- `衣装记述`：替代原普通描述。
- `命理影响`：展示 AI提示 的玩家可读版本。
- `价格`：灵石数值。

## 8. AI 读取闭环

### 前端写入

- 购买：仍写入 `道具.拥有[逻辑名] += 1`。
- 装备：仍写入 `道具.装备[目标].push(逻辑名)`。
- 卸下：从 `道具.装备[目标]` 移除逻辑名，并返还库存。
- 待处理交互：继续记录 `装备道具 / 卸下`，道具字段使用逻辑名。

### AI 读取

世界书/变量规则应约束 AI 每回合读取：

1. `系统.场景上下文.在场NPC`。
2. 当前在场 NPC 的 `道具.装备[NPC名]`。
3. `道具.装备.玩家`。
4. `系统.待处理交互` 中本回合新增的装备/卸下事件。

### 离场再入场

- NPC 离场时不清空 `道具.装备[NPC名]`。
- NPC 再次进入 `在场NPC` 后，角色卡装备栏继续显示其穿着。
- AI 描写再入场时应参考已有装备，不需要每次机械复述，但关键衣装需影响动作、神态或旁人反应。

## 9. 实施边界

- 本阶段不直接改 `名称`，避免破坏旧存档。
- 本阶段不新增复杂换装立绘，仅先完成文本、数据、展示与 AI 读取闭环。
- 本阶段不改变购买/装备/卸下核心生命周期，只补充显示名与分类元数据。
- 后续实现必须同步测试：商城展示、背包显示、角色卡装备显示、牝奴服目标限制、AI 世界书规则。
