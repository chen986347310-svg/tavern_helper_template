合欢宗·状态栏法器美化方案
核心设计哲学
法器化：圆环不再是进度条，而是 NPC 的“命魂星盘”。

生命力：所有状态变更必须是柔和的、呼吸状的，严禁数字硬跳。

去边界化：能用色块明暗区分的决不用线条；能用半透明流光暗示的绝不用实线框。

1. 缩小状态：从“小圆圈”到“命魂显影”
为了解决缩小后“难看”的问题，我们引入两个新的设计概念：“专属色晕”与“命魂呼吸”。

视觉重构 (Visual Spec)
取消实线：彻底移除缩小状态下的实线圆环。

命魂专属色 (Soul Color)：当卡片缩小，好感度（Favorability）不再以进度条形式存在，而是转化为一层围绕 NPC 头像的半透明柔光光晕。

光晕颜色必须匹配 NPC 的专属色：

白芷: #a8c4e0 (冷金)

纪兰: #b088d4 (紫金)

沈月秋: #d46048 (赤金)

光晕强度 (Glow Intensity)：光晕的透明度和扩散半径直接映射好感度数值。

0 好感：几乎透明，无光晕。

100 好感：柔和但清晰的常驻柔光。

交互动效：命魂呼吸 (Soul Breath)
常驻状态：光晕执行极慢（10s 周期）的 cubic-bezier(0.4, 0, 0.2, 1) 呼吸动画，模拟命魂的跳动。

状态变更（触发）：

当好感度增加时，专属光晕瞬间爆发（Expolsion）一下，向外扩散，随后缓缓收拢。

这个爆发的半径取决于数值增加的幅度。

SCSS


// CSS 伪代码实现缩小状态光晕
.npc-disk-collapsed {
  box-shadow: 0 0
    calc(var(--fav-value) * 0.2px + 5px) // 扩散半径随数值改变
    rgba(var(--soul-color-rgb), calc(var(--fav-value) * 0.0035 + 0.1)); // 透明度随数值改变
  transition: box-shadow 0.6s var(--hh-easing-rescale);
  animation: hh-breath 10s infinite alternate;
}
2. 展开状态：从“双进度条”到“阴阳命轮”
展开后的画面，通过引入“太极阴阳”和“专属图腾”来提升法器感。

视觉重构 (Visual Spec)
阴阳双环：

内环（阴环/侵蚀环）：锁定为胭脂色（#9C2C31，如沈月秋卡片的红线）。代表攻略/道心侵蚀。

外环（阳环/灵犀环）：锁定为 NPC 专属色（如沈月秋的赤金 #d46048）。代表好感。

环上图腾 (Disc Totem)：取消纯色的圆环。圆环本身由极其微小的、半透明的图腾残片（如合欢花辦、符文）拼接而成。随着进度填充，图腾从灰色逐渐亮起。

去数字化：严禁在圆环内或周遭显示数字（如 85/100）。

交互动效：湮灭与同化 (Assimilation)
常驻状态：太极盘中心执行慢速（20s）自转。

状态变更（触发）：

当好感或攻略值改变时，圆环上的图腾如墨迹一般扩散（Dispersion）填充，而不是直线的进度增加。

【终极动效：沦陷】：当攻略值达到 100 时，内圈红环（阴环）彻底亮起，并与外圈专属色环产生同化（Merging）动画，两环合一，最终变为一圈常驻的、具有流动感的深红色流光。象征道心崩塌，彻底沦陷。

既然是酒馆对话系统，我们可以让玩家更有目的性地点击状态栏，而不仅仅是查看数值。我们建议在 MD 文件中增加以下“风声引导层”。

视觉呈现：当状态栏内有风声情报（Rumor）更新时，状态栏容器周遭执行极轻微的涟漪（Ripple）动画。

交互体验：玩家悬停（Hover）在涟漪区域，UI 边缘滑出一个极小的、水墨渐变边缘的提示面板，显示当前最新的场景线索：

◎ 风声: 药庐丹炉彻夜未熄，白芷似乎屏退了所有杂役。

价值：让点击状态栏变成一个具有剧情驱动力的行为，强化“社交博弈”的代入感。

5. 命盘灵识窥探系统 (Soul Spy System)
5.1 窥心诱导交互：灵气潮汐 (Spiritual Tide)
为了诱导玩家点击，展开后的命盘绝不能是一个死板的圆环。我们需要用“数值不稳”或“灵气溢出”的视觉特征，暗示玩家这里“有机可乘”。

常驻状态（无情报）：命盘双环保持慢速自转，环上的图腾残片呈现半透明的暗金色或专属色。

诱导异动（有心声可窥探时）：当 AI 判定该场景下该 NPC 有“心里话”可以被抓取时，命盘会触发 “灵气潮汐” 动效：

光流紊乱：双环的转速开始忽快忽慢，原本首尾相接的 SVG 描边出现间歇性的断裂、重组。

字符悬浮：圆环周围会隐隐浮现出一些极淡、极小的、乱码状的古字残片（利用 CSS opacity 在 10% 到 40% 之间随机闪烁），像是一层由于心防不稳而外泄的“精神碎片”。

悬停反馈（Hover）：当玩家鼠标悬停在紊乱的命盘上时，指针转化为“手掌”形态，整个命盘瞬间向内微微收缩（scale(0.98)），并伴随极其低沉的琴音共振。这在心理学上会产生强烈的“压迫感与破局欲”，诱导玩家推门入局、一探究竟。

5.2 核心交互状态：灵识锁定 (The Thread of Soul)
点击命盘后，玩家的行为需要被准确记录到你架构中的 待处理交互（Pending Action）中，并触发视觉锁定。

红线缠绕：点击命盘的瞬间，紊乱的光流平息，取而代之的是一条极细的、猩红色的 “因果线”（利用 SVG 线条动画）从太极盘核心射出，死死缠绕在双环的中轴线上。

挂载待处理标识：卡片底部轻柔浮现四个竖排的暗色小字：【灵识窥伺】，代表当前动作已写入暂存，将在下一回合发送给 AI 进行判定。

撤销操作：再次点击命盘，红线寸寸断裂溃散，交互状态清空，允许玩家反悔。

5.3 结算分歧：因果回写与心房反震 (The Backlash)
当玩家提交回合后，下一楼层的 AI 读取 待处理交互: 灵识窥探。基于 NPC 的道心强弱与当前好感度，AI 回写变量会产生两种视觉结果：

结果 A：窥心成功 (Whisper Revealed)
视觉表现：下一回合卡片刷新时，缠绕在命盘上的红线化为一缕轻烟散入卡片背景。

心里话注入：NPC 头像下方或侧边，原本空白的环境留白处，以慢速逐渐显影出该 NPC 的深层心声。字色采用比正文叙事更暗的灰色或冰蓝色，营造出“窃听”的私密感。

结果 B：心房反震（Backlash - 道心防御）
这是你提到的高风险惩罚机制。当高道心 NPC 察觉并反击时，UI 必须给玩家极强的“挫败感与痛觉反馈”。

法器碎裂动效（The Shatter）：

在下一回合收到 AI 的反震 Patch 时，原本缠绕命盘的红线骤然变为刺眼的亮红。

命盘（双环 SVG）产生剧烈的震动（利用 CSS keyframes 编写高频无规则位移），随后伴随一声刺耳的清脆玉碎声效，双环上的图腾残片直接炸开、崩飞淡出（Disintegrate）。

数值扣减（Blood Leak）：

命盘炸开后，好感度环（阳环）以肉眼可见的速度往回倒退。倒退留下的空白轨道，会闪烁着不祥的暗红色。

头像周围原本缩小时好看的“专属色晕”瞬间熄灭，卡片全局色调在 1s 内被降饱和度，呈现出一片灰败之色，象征神魂受损。

防备文案与乱码：

心里话位置不会显示真实心声，而是被一串毫无规律的乱码古字（如 纟纟... \ 露...）填满，最后乱码淡出，变为一行冷冰冰的提示：【神魂受震，心防彻底锁闭】。

6. 修订后的系统验收红线 (UX Section)
为了确保这个博弈系统不变味，在验收时必须严格卡死以下红线：

严禁提示明示：界面绝对不允许出现类似“点击此处窥听心里话，有概率反噬”的文字提示。所有的危险与机遇，必须完全依靠命盘的光流紊乱（诱导）和红线缠绕（锁定）由玩家自己去悟。

惩罚即时同步：一旦 AI 发回反震 Patch，好感度圆环的倒退和卡片的灰败动画必须同时发生，利用视觉的“痛感”让玩家对高道心 NPC 产生敬畏。

状态闭环：反震后，该 NPC 的命盘在当前场景内必须保持“死寂状态”（不再有自转，不再有潮汐），直到下一次场景重排或好感度通过正向互动修复。
