# P2 牝奴期 UI/UX 设计需求

日期：2026-05-25

## 1. 设计目标

P2 牝奴期界面要表达的核心体验是：玩家已经不再是攻略者，而是被合欢宗纳入秩序、被日课驱动、被牝印命令、被 NPC 支配、被宗门持续凝视和议论的中心。

界面第一眼应让玩家感到：

- 我正在被牝印控制。
- 今天有日课要执行。
- 当前有人拥有我的处置权。
- 世界正在低声议论我。
- 我的身体、服装、道具、羞名和调教记录都会留下痕迹。

P2 不是结局展示页，也不是普通状态面板。它应该是一个“牝奴期主控台”。

## 2. 当前界面问题

现有 P2 页面主要展示：

- 堕落度阶段
- 淫纹数量
- 牝阴决层数
- 泌乳 / 肛门 / 憋尿改造
- 已装备拘束法器

这些适合作为状态信息，但不足以支撑 P2 作为第二主玩法。

当前缺失：

- 当前日课
- 当前支配者
- 当前牝印命令
- 命令强度
- 今日调教次数
- 最近调教结算
- 宗门低语 / 听风羞名
- 烙名痕迹
- 可点击承接的 P2 风声入口

## 3. 信息架构

P2 首页建议分为 5 个主要区域。

### 3.1 牝印核心区

用途：页面主视觉，表达“身体正在被牝印支配”。

应展示：

- 堕落度
- 牝阴决层数
- 当前命令强度
- 当前命令短句
- 牝印状态，例如沉寂、发热、强制、回响、烙名

视觉建议：

- 不要做普通数值卡片。
- 可以用圆环、印纹、花纹、阵纹、身体印记形态表达。
- 堕落度和牝阴决可以围绕同一个“牝印核心”组织。
- 命令强度可以用印纹亮度、裂纹、朱批、铃痕表达。

### 3.2 今日日课区

用途：告诉玩家当前时辰世界正在如何使用她。

应展示：

- 当前时辰
- 当前日课名称
- 日课方向：验身、服侍、试药、调教、寝役、登记、示众等
- 待执行日课
- 日课是否已被风声或支配者改变

示例信息：

```text
晨时 · 验身
执事殿今日要复查牝印反应。
```

```text
酉时 · 听风廊示众
昨日铃声失序的风声已传开。
```

视觉建议：

- 可以做成“日课卷轴”或“执事名册”。
- 不要像现代日程表。
- 当前日课要比未来日课更突出。
- 被羞名影响的日课可以有朱批、划痕、铃印等视觉标记。

### 3.3 当前支配者区

用途：表达“现在谁拥有玩家的处置权”。

应展示：

- 当前支配者
- 支配者风格
- 支配关系状态
- 支配次数或近期支配痕迹
- 候选支配者或下一位可能介入者

五名 NPC 的方向：

- 白芷：温柔而坚定，关怀包裹支配。
- 苏芸：试药、挑逗、身体反应观察。
- 纪兰：记录、训诫、规则化调教。
- 沈月秋：秩序、权威、公开安排。
- 柳素衣：掌门注视、牝印命令、归属重分配。

视觉建议：

- 不要只是头像卡。
- 要让玩家感觉“被某人看着”。
- 可以使用目光、名牌、朱批、命令签、牵丝线等隐喻。
- 当前支配者要和日课、命令、羞名联动。

### 3.4 听风羞名区

用途：P2 风声系统。让玩家感到自己的失态、日课、调教后果正在宗门中流传。

应展示 1-3 条羞名风声。

每条包含：

- 凝视来源：弟子低语、执事记录、NPC目光、掌门注视、牝印回响
- 地点
- 羞名等级：微闻、传开、挂牌、示众、烙名
- 风声短句
- 是否可点击承接

示例：

```text
◎ 弟子低语 · 听风廊 · 微闻
她走过时铃声乱了一拍，像是牝印先替她认了错。
```

```text
◉ 执事记录 · 经阁 · 挂牌
今日晨课缺一项验身记录，纪兰已在名册旁留了朱批。
```

```text
◆ 掌门注视 · 偏殿 · 烙名
柳素衣没有问她为何失态，只让人把她的名字移到今晚的寝役册上。
```

交互要求：

- 点击羞名风声后，不是接任务，而是“被风声牵引”。
- 点击后应进入待处理状态，等待下一楼层 AI 承接。
- 视觉上可以从低语变成锁定、朱批、牵引、被点名。

视觉建议：

- 不要做成任务列表。
- 应像低语流、风铃声、纸页边批、名册暗纹。
- 文字可横向流动或轻微漂移，但不能影响可读性。
- 活跃羞名可以有微弱涟漪，呼应状态栏风声提示。

### 3.5 烙名痕迹区

用途：展示玩家近期获得的羞名标签，让调教后果有记忆点。

标签示例：

- 铃声失序
- 验身迟疑
- 牝印发热
- 药香外泄
- 目光回避
- 寝役留名
- 示众未毕
- 名册朱批
- 体香难遮
- 命令迟滞

视觉建议：

- 不要做现代徽章。
- 更像朱批、铃痕、牝印残光、名册烙印。
- 可以按新旧、强弱、来源排列。
- `烙名` 等级的标签要比普通标签更持久、更重。

## 4. 页面优先级

P2 首页第一屏优先级：

1. 牝印核心
2. 当前日课
3. 当前支配者
4. 听风羞名
5. 烙名痕迹
6. 身体改造
7. 拘束法器
8. 最近调教记录

现有“身躯改塑”和“拘束法器”不要删除，但它们不应再是页面主角。它们应该成为下半部分的状态与后果展示。

## 5. 交互状态

设计师需要覆盖这些状态：

- 无当前命令
- 有当前命令
- 命令强度升高
- 当前日课普通
- 当前日课被羞名风声改变
- 当前支配者为空
- 当前支配者存在
- 羞名风声为空
- 羞名风声 1 条
- 羞名风声 3 条
- 羞名风声被点击锁定
- 烙名痕迹为空
- 烙名痕迹多条
- 改造进度未完成 / 部分完成 / 全部完成
- 拘束法器为空 / 已装备

## 6. 视觉气质

关键词：

- 牝印
- 朱批
- 风铃
- 低语
- 名册
- 牵丝
- 合欢花
- 烙痕
- 牝纹
- 暗金
- 浅红
- 墨紫
- 玉白
- 微光

避免：

- 现代任务列表
- 大面积纯粉色
- 卡片堆卡片
- 明亮可爱风
- 过度游戏 HUD
- 直接用“任务 / 成就 / 通知 / 领取奖励”等现代词
- 只堆露骨文字，不做世界感表达

## 7. 推荐布局

推荐方案：纵向主控台。

```text
[牝印核心区]
堕落度 / 牝阴决 / 当前命令强度

[今日日课区]
当前时辰 + 当前日课 + 待执行日课

[当前支配者区]
当前支配者 + 支配风格 + 当前处置权

[听风羞名区]
1-3 条宗门低语，可点击承接

[烙名痕迹区]
近期羞名标签

[身体改造 / 拘束法器 / 最近调教记录]
下半部分折叠或弱化展示
```

## 8. 状态栏需求

P2 状态栏需要从现在的“堕落度 + 牝阴决”升级为：

- 牝印核心小图标
- 堕落度
- 牝阴决层数
- 当前时辰
- 若有活跃羞名风声，出现低语涟漪
- 若有当前命令，牝印图标发热或轻微闪烁

注意：状态栏不要塞太多文字。它只做全局气氛和关键状态提示。

## 9. 移动端 / 小容器要求

当前应用是窄屏卷轴式容器，所以设计必须适合小宽度。

要求：

- 单列布局优先。
- 关键文字不能横向溢出。
- 风声短句最多两行，超出渐隐或折叠。
- 按钮点击区域要足够大。
- 不要依赖 hover 才能理解。
- 动画要轻，不影响阅读。
- 信息密度要高，但不能像后台面板。

## 10. 设计交付物

请设计师输出：

1. P2 首页完整视觉稿。
2. P2 状态栏视觉稿。
3. 听风羞名组件的 3 个状态：空、普通、有锁定。
4. 当前支配者组件的 3 个状态：无支配者、普通支配、柳素衣注视。
5. 牝印核心组件的 3 个状态：沉寂、发热、强制命令。
6. 烙名痕迹组件的空状态和多标签状态。
7. 移动窄屏适配稿。
8. 颜色、字体、图标/纹样说明。

## 11. 验收标准

设计稿合格标准：

- 第一眼能看出这是 P2，不是 P1 换皮。
- 第一屏能看出当前日课和当前支配者。
- 能明显感到玩家正在被凝视、记录、议论。
- 风声系统在 P2 中不再像普通线索，而像羞名低语。
- 视觉有世界观，不像现代任务面板。
- 牝印、羞名、日课、支配者、改造状态之间有层级关系。
- 不依赖大段说明文字解释玩法。
- 小屏可读，文本不溢出，不遮挡核心内容。

## 12. 设计边界

第一版设计不需要覆盖完整 P2 第二主路线树，也不需要做复杂舆论传播模拟。重点是让 P2 首页完成最小闭环：玩家看到牝印核心、当前日课、当前支配者、听风羞名和烙名痕迹，并能理解羞名风声会把自己牵引到下一次调教事件。
