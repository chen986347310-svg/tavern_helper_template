# 服装商城重构映射表 v3：AI叙事提示增强版

> 日期：2026-05-24  
> 范围：服装显示名、短提示、AI叙事规则、场景变化、角色适配边界。  
> 状态：设计草案，基于 `服装商城重构映射表v2-香艳暗示版.md` 增强 AI 可读性。

---

## 1. v3 核心结论

服装不能只作为“背包里的一行文字”。在 AIRP 对话式游戏中，服装应成为 AI 每回合可读取的叙事锚点：

1. **前端负责显示与交互**：商城、背包、角色卡显示 `显示名` 与 `AI短提示`。
2. **MVU 负责稳定事实源**：装备状态仍只写入 `道具.装备[target]`，使用原 `名称` 作为逻辑 key。
3. **世界书负责叙事解释**：AI 根据逻辑 key 查服装叙事规则，决定何时描写、如何描写、何时不写。
4. **不机械复述**：AI 不应每回合重复“某某穿着某衣”，而应让衣装影响动作、气氛、旁人目光、灵气反应与角色自我意识。
5. **离场再入场保留**：只要 `道具.装备[NPC名]` 或 `道具.装备.玩家` 仍存在，再入场时 AI 与 UI 都应继续承接穿着。

## 3. 通用 AI 叙事规则

### 3.1 每回合读取顺序

AI 在写正文前应按顺序读取：

1. `系统.场景上下文.地点 / 子区域 / 公开度 / 在场NPC`。
2. `道具.装备.玩家`。
3. 每个在场 NPC 的 `道具.装备[NPC名]`。
4. `系统.待处理交互` 中本回合新增的 `装备道具 / 卸下`。
5. 对应服装的 `服装叙事规则`。

### 3.2 触发原则

服装描写只在以下事件发生时自然触发：

- 角色入场、离场、转身、低头、坐下、起身。
- 奔走、施法、打扫、炼丹、练功等身体动作。
- 被玩家注视、被 NPC 点名、被旁人议论。
- 场景公开度变化：公开 → 半私密 → 私密 → 禁地。
- 心绪变化：羞怯、克制、动摇、沉沦、反抗。
- 新装备或卸下后的下一楼层承接。

### 3.3 不触发原则

AI 不应为了“记得衣服”而每回合硬写。以下情况应弱化或跳过：

- 当前正文重点是重大剧情、战斗、变量更新或场景转换。
- 角色不在 `在场NPC` 中，且没有被回忆/传闻/风声提及。
- 服装已卸下，`道具.装备[target]` 不再包含该逻辑名。
- 角色设定或当前情境不适合成人向承接。

## 6. 世界书规则草案

```yaml
服装装备读取规则:
  读取顺序:
    - 系统.场景上下文.公开度
    - 系统.场景上下文.在场NPC
    - 道具.装备.玩家
    - 道具.装备[在场NPC]
    - 系统.待处理交互
  描写原则:
    - 不要每回合机械复述装备名。
    - 当角色入场、移动、被注视、心绪波动、施法或场景公开度变化时，才自然带出服装影响。
    - 装备存在于 MVU 中即视为持续穿着，NPC 离场不清除；再入场后仍可承接。
    - 卸下后不得继续描写该服装，除非作为回忆或风声提及。
  适配边界:
    - 成人/高阶 NPC 可承接更强暗示。
    - 低龄或不适配角色只写衣料、法纹、身份压迫、尴尬动作，不写露骨身体细节。
    - 玩家牝奴服只读取 道具.装备.玩家，不允许写给 NPC。
```

## 9. 五位 NPC 仙奴状态专属服装

### 9.1 设计定位

五位核心 NPC 在攻略完成、命契成立或状态进入“仙奴/已完成”后，应各自解锁一套专属终局服装。该服装不是普通商品，而是角色线完成后的视觉奖章：

- **解锁条件**：对应 NPC `状态 = 已完成` 或 `攻略值 >= 100`。
- **获取方式**：解锁后进入商城 `命契层`，玩家仍需消耗高额灵石购买。
- **装备对象**：只能装备给对应 NPC，不允许跨角色装备。
- **AI用途**：作为该 NPC 仙奴化后的身份锚点；离场再入场时，如果仍装备，应自然承接其服装、姿态和身份变化。
- **UI用途**：角色卡装备栏显示专属显示名；商城详情标注“命契专属”。

### 9.4 商城与背包规则

- 商城 `命契层` 中，专属服装默认隐藏。
- 当对应 NPC `状态 = 已完成` 或 `攻略值 >= 100` 时，对应专属服装显示。
- `canBuy` 必须校验对应 NPC 是否完成，不能只校验灵石。
- 背包装备目标中，专属服装只显示对应 NPC。
- 如果玩家尝试装备给其他目标，前端应拒绝并提示 `命契不合，此衣不认其主`。
- 卸下后库存返还，专属服装可再次装备给原 NPC。
