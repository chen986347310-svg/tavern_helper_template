# 灵石经济重平衡方案 v2

## 1. 背景

旧版灵石经济建立在较小商城规模上：服装约 25 件、禁器约 12 件、永久丹药约 6 枚，旧 PRD 估算全买约 47,600 灵石，常规全攻略收入约 23,460 灵石，设计目标是“玩家必须做选择”。

当前商城已经扩容：服装、禁器、丹药、牝奴道具和特殊道具共同构成更大的游玩池。若继续沿用旧收入公式，玩家会长期看得到但买不起，商城从“选择压力”变成“橱窗压力”。

本方案目标不是让玩家无脑全买，而是让玩家在攻略期经常能买到有趣物品，并在中后期能围绕偏爱的 NPC 建立完整服装、禁器、丹药组合。

## 2. 当前消费池体检

按当前前端数据统计：

| 分类 | 数量 | 总价 |
| --- | ---: | ---: |
| 禁器 | 24 | 48,550 |
| 丹药 | 40 | 129,400 |
| 服装 | 31 | 83,800 |
| 时间延长 | 1 | 10,000 |
| 牝奴道具 | 4 | 11,000 |
| 不含改变阵法的可玩消费 | 100 | 282,750 |
| 改变阵法 | 1 | 500,000 |

按好感/状态门槛统计，不含改变阵法：

| 门槛 | 数量 | 总价 |
| --- | ---: | ---: |
| 0 | 14 | 26,150 |
| 30 | 19 | 19,900 |
| 50 | 17 | 26,700 |
| 70 | 16 | 36,500 |
| 90 | 9 | 36,800 |
| 100 | 21 | 125,700 |
| 牝奴专属 | 4 | 11,000 |

现有常规收入约 23,460 灵石，只覆盖新版可玩消费池约 8.3%。这已经明显偏紧。

## 3. 设计目标

1. 开局可买：初始阶段应能购买 1~3 件低价物品，尽快让玩家体验商城反馈。
2. 中期常买：每完成一个 NPC 的阶段推进，玩家应能买一组小组合，例如一件服装、一件禁器、一枚丹药。
3. 后期有规划：命契层、仙奴丹和专属服仍然昂贵，玩家不能一口气全收。
4. 真结局仍有重量：`改变阵法` 继续保持 500,000 灵石，并主要依赖柳素衣完成奖励。
5. 世界观自洽：收入不只是“刷钱”，而是来自合欢宗资源配给、双修收益、风声追查、归契赏赐和宗门账册。

## 4. 推荐方案：中等偏大方

推荐把攻略期常规可得灵石从约 23,460 提升到约 120,000~160,000。

这个区间能覆盖新版可玩消费池的 42%~57%，足够让玩家体验大部分感兴趣的中低阶系统，但仍无法无脑买空商城。高阶专属服、仙奴丹、命契禁器和时间延长仍然需要取舍。

### 4.1 初始灵石

建议 `系统.灵石` 初始值从 0 改为 2,000。

理由：

- 2,000 可以购买若干低阶禁器、服装或温药。
- 不足以跳过中期经济规划。
- 符合“外门资源配给”或“合欢宗试炼启动资金”的世界观。

可选范围：

- 保守：1,500。
- 推荐：2,000。
- 大方：3,000。

### 4.2 基础收入公式

现有公式：

```text
灵石增量 = NPC境界系数 × 攻略值增量
```

建议调整为：

```text
灵石增量 = NPC境界系数 × 攻略值增量 × 4 + 阶段赏赐
```

境界系数保留不变：

| NPC | 系数 |
| --- | ---: |
| 白芷 | 10 |
| 苏芸 | 20 |
| 纪兰 | 40 |
| 沈月秋 | 60 |
| 柳素衣 | 100 |

这样能保留“高阶 NPC 更值钱”的原设计，同时把基础收益提升到可支撑新版商城的水平。

### 4.3 阶段赏赐

每名 NPC 首次达到攻略值节点时，获得一次宗门赏赐。

| 攻略值节点 | 奖励 | 叙事含义 |
| --- | ---: | --- |
| 30 | 800 | 初入心防，账册记功 |
| 50 | 1,500 | 关系动摇，资源追加 |
| 70 | 2,500 | 心防深陷，内库拨款 |
| 90 | 4,000 | 命契将成，宗门重赏 |
| 100 | 6,000 | 攻略完成，归契赏赐 |

单 NPC 全节点奖励 14,800，五名 NPC 合计 74,000。

这部分奖励的价值在于稳定托底：即使 AI 单次给的攻略值增量较小，玩家也能在阶段推进时获得明显反馈。

### 4.4 风声追查奖励

追查风声成功后可获得小额灵石或折扣型收益。

推荐规则：

| 风声结果 | 奖励 |
| --- | ---: |
| 普通线索 | 300~600 |
| 重要线索 | 800~1,200 |
| NPC 私密线索 | 1,500~2,000 |

注意：风声奖励应由 AI 在下一楼层根据叙事结果回写，不由前端点击时即时发放。

设计目的：

- 鼓励玩家使用动态场景和风声系统。
- 让非 NSFW 推进也能获得资源。
- 避免玩家为了赚钱只重复同一类互动。

### 4.5 每日配给

每日晨时可获得基础配给，建议 300~500 灵石。

推荐：每天 400 灵石，30 天理论上限 12,000。

设计含义：

- 合欢宗对试炼弟子发放基本资源。
- 防止玩家在早期因一次购买失误完全卡死。
- 数量不大，不会破坏中后期规划。

实现边界：

- 需要防止同一天重复领取。
- 若当前变量没有“已领配给日期/天数”字段，第一版可以先不落代码，只写入 AI 规则，由 AI 在过夜时追加灵石。

## 5. 预期收入结构

推荐方案下，攻略期可得灵石约为：

| 来源 | 估算 |
| --- | ---: |
| 基础攻略收益 ×4 | 约 93,840 |
| 阶段赏赐 | 约 74,000 |
| 每日配给 | 0~12,000 |
| 风声追查 | 约 5,000~20,000 |
| 柳素衣完成奖励 | 1,000,000，仅服务真结局/终局消费 |

不含柳素衣 1,000,000 奖励，常规攻略期总收入约 170,000~200,000 上下。实际收入会因玩家攻略速度、是否追查风声、AI 回写力度而浮动。

这个数值可以支撑：

- 低中阶服装、禁器、丹药较自由地购买。
- 为 1~2 名偏爱 NPC 配出完整高阶组合。
- 仍无法买空 282,750 的可玩消费池。
- `改变阵法` 仍必须依赖柳素衣完成奖励或后期特殊剧情。

## 6. 消费侧是否降价

不建议大规模降价。

理由：

- 价格已经承担阶层叙事：命契层、仙奴丹、高阶禁器应该有重量。
- 降价会削弱商城“越深入越昂贵”的欲望曲线。
- 当前问题主要是收入旧了，不是商品定价整体错了。

建议只微调两个位置：

1. 门槛 0 的入门商品可以保证至少 6~8 件价格低于 1,000。
2. 仙奴丹不必降价，仙奴阶段本来就该更夸张、更昂贵。

## 7. 世界观解释

灵石不是单纯货币，而是合欢宗对“可利用关系、双修资源、心防突破、风声线索”的量化结算。

推荐叙事口径：

- 攻略值推进：说明目标心防松动，宗门账册认为玩家“可继续投入”。
- 阶段赏赐：由内库、药庐、经阁或执事堂发放。
- 风声追查：玩家提供可交易的情报、把柄或修行线索。
- 每日配给：试炼弟子基础修行资源。
- 柳素衣完成奖励：掌门命契完成后，宗门资源权限打开，真结局门票才现实。

## 8. 需要修改的文件

若进入实现，建议修改：

暂不建议第一版新增复杂字段，例如“今日是否已领配给”。如果要做每日配给自动化，再单独开一轮 schema 设计。

## 9. 测试建议

第一版实现后的测试重点：

1. `calculate灵石获取('白芷', 10)` 从 100 调整为 400。
2. 五名 NPC 系数仍保持相对差异。
3. 商城购买扣费逻辑不变，不能因收入调整导致负数。
4. `initvar.yaml` 和 schema dump 后仍合法。
5. 世界书规则中不允许 AI 重复执行前端已完成扣费。

## 10. 推荐落地顺序

1. 先改基础倍率与初始灵石，这是最小闭环。
2. 再改世界书灵石经济说明，让 AI 按新口径发放阶段赏赐和风声奖励。
3. 跑 `guards.test.ts` 与完整 `npx vitest run`。
4. 如果实机后仍偏紧，再考虑每日配给自动化字段。
