## 三、AIRP 闭环审计

### 3.1 购买物品 — CLOSED

```
ShopPage.buyItem() → usePendingAction.记录购买物品()
  → 系统.待处理交互[].类型='购买物品' + 叙事元数据
  → pendingActionPrompt.ts 注入 hehuan-pending-action-summary (in_chat)
  → 世界书绿灯：道具AI承接闭环 / 特殊道具事件规则
  → AI 输出 <UpdateVariable><JSONPatch>
  → MVU 回流 → store.data 响应式更新 → ShopPage 重渲染
```

### 3.2 装备/卸下 — CLOSED

```
BackpackPage → usePendingAction.记录装备道具() / 记录卸下道具()
  → 系统.待处理交互 + 器阶/作用部位元数据
  → pendingActionPrompt.ts 注入 + 扫描禁器叙事规则/服装叙事规则
  → AI JSONPatch → MVU 回流 → BackpackPage 重渲染
```

### 3.3 追查风声 — CLOSED

```
HomePage → usePendingAction.记录追查风声(rumor)
  → 系统.待处理交互 + 地点/子区域/风声ID/故事钩子/在场NPC
  → pendingActionPrompt.ts 注入场景遭遇指令
  → AI 更新 场景上下文 + 当前追查风声ID + 风声列表 → 清空队列
  → MVU 回流 → HomePage 重渲染
```

### 3.4 灵识窃取 — BROKEN (by design)

旧主动灵识窃取已在 v4 中被被动 `心声探测态` 系统取代：

- `usePendingAction.记录灵识窃取` 仍导出但无 .vue 页面调用
- `HomePage.vue` 主动清除队列中的灵识窃取条目
- `guards.ts` 的 `perform灵识窃取` 全系函数（4 个）均为死代码
- 世界书保留旧队列兼容承接规则（变量更新规则.yaml line 341）

**判定：设计意图明确，旧代码为向后兼容保留，非 bug。**

### 3.5 队列清空机制 — CLOSED (prompt-only, 无代码兜底)

| 层 | 机制 |
|----|------|
| 世界书 | 变量更新规则.yaml：必须同一次 JSONPatch 清空 |
| 动态注入 | pendingActionPrompt.ts：注入"承接后必须清空系统.待处理交互为[]" |
| 输出格式 | 变量输出格式.yaml：清空补丁格式示例 |
| 代码兜底 | validate.ts `ensureV4SystemFields`：仅补默认值，**不强制清空非空队列** |

**风险：若 AI 模型未输出清空补丁，残留交互将在下一楼层被重复处理。此为纯 prompt 工程约束，无程序化兜底。**

### 3.6 validate.ts 覆盖范围

已覆盖：v4 字段引导、类型修正、攻略链顺序、装备门槛、改变阵法守卫、数值范围钳制。

未覆盖：队列清空验证、叙事完整性验证、场景上下文更新验证。
