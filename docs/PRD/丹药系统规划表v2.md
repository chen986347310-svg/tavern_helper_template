# 丹药系统规划表 v2

日期：2026-05-24  
状态：规划稿，待实现

## 一、系统定位

丹药不是普通 buff，也不是商城里的简单消耗品。丹药是合欢宗用于改写身体底层反应、降低羞耻边界、固化攻略成果、服务仙奴调教的内服法门。

服装负责可见身份，禁器负责外置触发，丹药负责内部改写。丹药的叙事重点不是“吃了加属性”，而是：为什么合欢宗需要这颗丹药，谁会吃，为什么会吃，吃完后体态、生理、思维或性功能发生什么变化。

## 二、设计原则

1. 逻辑名保留旧存档兼容，显示名可更含蓄。
2. 丹药分临时、永久、仙奴三类，不再全部归入“永久丹药”。
3. 丹药必须和好感度或角色状态挂钩，不能玩家拿给 NPC 就默认吃。
4. 每枚丹药必须有设计驱动：它为哪类关系推进、身体改造或后期玩法服务。
5. 临时丹药负责制造场景和剧情失衡；永久丹药负责固化长期身体/思维变化；仙奴丹负责性服务、取悦、身体可玩乐化和归属展示。
6. 丹药效果不应只在性爱过程中出现。公开场景中的体香、步态、语气、衣料痕迹、被看见、被点名、被检查，同样是重要羞耻来源。
7. AI 承接时必须使用 `道具` 逻辑名作为变量主键；显示名、药阶、作用线、AI短提示只作为叙事辅助。

## 三、门槛规则

| 门槛 | 使用边界 |
|---|---|
| 0 好感 | 只能是可解释、低风险、像调息或温养的丹药。 |
| 30 好感 | 允许轻度羞耻、体态外显、情绪松动。 |
| 50 好感 | 允许身体敏感、梦境、乳房/肌肤/体液变化。 |
| 70 好感 | 允许明显合欢宗药性，泌乳、发情、器官反应、羞耻阈值改变。 |
| 90 好感 / 已完成 | 允许命契、归属、思维反射、不可逆痕迹。 |
| 仙奴状态 | 开放仙奴丹，不再伪装成普通调息药，直接服务性与身体可玩乐化。 |

## 八、实现建议

实现时建议分三步：

1. 数据层：新增 `临时丹药`、`永久丹药`、`仙奴丹` 三类结构，旧 `永久丹药` 名称保留逻辑兼容。
2. 商城/背包层：商城按丹药类型或药阶分组；背包使用时继续只对当前在场 NPC 显示可用目标。
3. AI 承接层：等服装、禁器、丹药三套数据都稳定后，统一补“道具叙事闭环”，让 AI 根据待处理交互、显示名、类型、作用线和 AI短提示承接。

本 PRD 只定义丹药系统规划，不直接要求本轮实现代码。
