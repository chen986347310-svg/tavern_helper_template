# 雌堕合欢宗发布维护与开发经验

日期：2026-05-26

本文件记录当前版本的权威发布口径、角色卡打包方式、CDN 缓存处理、入口主页问题修复、P1/P2 回归经验和后续维护红线。后续接手时先读本文件，再看具体 PRD 和计划记录。

## 1. 当前发布口径

- 主仓库：`https://github.com/chen986347310-svg/tavern_helper_template`
- 仓库可见性：公开仓库。`jsDelivr` 只能稳定读取公开仓库资源。
- 运行资源目录：`public/hehuan/`
- 当前角色卡运行 CDN：`https://cdn.jsdelivr.net/gh/chen986347310-svg/tavern_helper_template@main/public/hehuan`
- 不再使用本地 `5500` 端口作为交付入口。
- 不再使用独立 `hehuan-cloud-assets` 仓库路线。
- `testingcf.jsdelivr.net` 可用于部分第三方模板资源，但本项目运行资源已切到标准 `cdn.jsdelivr.net`，原因是 `testingcf @main` 在 2026-05-26 出现旧缓存无法及时刷新的问题。

当前需要发布到 CDN 的文件：

| 文件 | 作用 |
| --- | --- |
| `public/hehuan/entry-home.html` | 角色卡入口主页 |
| `public/hehuan/status/index.html` | 状态栏前端 |
| `public/hehuan/scripts/mvu.js` | MVU 外链入口 |
| `public/hehuan/scripts/var_structure.js` | 变量结构注册脚本 |
| `public/hehuan/scripts/backend_validate.js` | 后端校验、动态提示、运行时闭环脚本 |

## 3. 角色卡打包经验

### 3.1 JSON 与 PNG

`src/雌堕合欢宗/index.yaml` 中：

```yaml
头像: null
```

当头像为空时，`tavern_sync.mjs bundle 雌堕合欢宗` 会生成角色卡 JSON 内容。但当前工具仍会按 `导出文件路径: dist/雌堕合欢宗` 输出到 `dist/雌堕合欢宗.png`。这个文件不是 PNG，文件头是 `{`，直接按图片导入酒馆会失败。

正确交付方式：

1. 打包。
2. 将 `dist/雌堕合欢宗.png` 改名为 `dist/雌堕合欢宗.json`。
3. 用 JSON 导入酒馆。

如果以后要交付 PNG 角色卡，必须把 `头像` 改成真实图片路径，例如：

```yaml
头像: 界面/assets/avatars/柳素衣.png
```

再打包。不要把 JSON 内容伪装成 PNG 发给玩家。

### 3.2 导入后入口页不生效

若入口主页显示出来但点击入口无反应，优先检查：

- `entry-home.html` 是否仍有旧代码 `document.currentScript.closest(...)`。
- CDN 是否拿到新版 `entry-home.html`。
- 酒馆环境是否暴露 `setChatMessages`。

已修复后的入口页逻辑：

- 不依赖 `document.currentScript.closest(...)`。
- 通过 `document.querySelectorAll('[data-hh-entry-home]')` 找最后一个入口根节点。
- 点击时再从 `globalThis`、`parent`、`top` 多路径解析 `setChatMessages`。
- 无 API 时必须显示文字提示，不能静默无反应。

本地最小验证信号：

- 四个入口按钮都能绑定点击事件。
- 第四入口应调用：

```js
setChatMessages([{ message_id: 0, swipe_id: 3 }], { refresh: 'all' })
```

## 4. CDN 与缓存刷新

### 4.1 jsDelivr purge

刷新缓存使用：

`https://www.jsdelivr.com/tools/purge`

网页工具只接受标准 `cdn.jsdelivr.net` URL。不要填 `testingcf.jsdelivr.net`，否则会提示 `Invalid URLs!`。

可一次填入：

```text
https://cdn.jsdelivr.net/gh/chen986347310-svg/tavern_helper_template@main/public/hehuan/entry-home.html
https://cdn.jsdelivr.net/gh/chen986347310-svg/tavern_helper_template@main/public/hehuan/status/index.html
https://cdn.jsdelivr.net/gh/chen986347310-svg/tavern_helper_template@main/public/hehuan/scripts/mvu.js
https://cdn.jsdelivr.net/gh/chen986347310-svg/tavern_helper_template@main/public/hehuan/scripts/var_structure.js
https://cdn.jsdelivr.net/gh/chen986347310-svg/tavern_helper_template@main/public/hehuan/scripts/backend_validate.js
```

刷新成功时，状态接口会显示各路径 `providers.CF = true`、`providers.FY = true`。

### 4.2 验证 CDN 是否更新

```powershell
$url = 'https://cdn.jsdelivr.net/gh/chen986347310-svg/tavern_helper_template@main/public/hehuan/entry-home.html'
$text = (Invoke-WebRequest -UseBasicParsing -Uri $url -TimeoutSec 30).Content
[pscustomobject]@{
  stableRootSelector = $text.Contains("document.querySelectorAll('[data-hh-entry-home]')")
  oldCurrentScriptCode = $text.Contains('document.currentScript.closest')
  resolveApi = $text.Contains('const resolveApi = () =>')
  length = $text.Length
}
```

预期：

- `stableRootSelector = True`
- `oldCurrentScriptCode = False`
- `resolveApi = True`

## 5. 正则与远程页面包装

入口主页和状态栏均由角色卡正则替换为 HTML 包裹页，再通过 jQuery `.load()` 拉取远程文件。保持这个结构：

```html
<body>
  <script>
    $('body').load('https://cdn.jsdelivr.net/gh/chen986347310-svg/tavern_helper_template@main/public/hehuan/entry-home.html')
  </script>
</body>
```

状态栏同理，但状态栏包裹页还包含旧楼层 iframe 隐藏逻辑：

- 给当前 iframe 标记 `data-hehuan-status-frame="1"`。
- 找到旧 iframe 后设置 `display=none`、`height=0`、`margin=0`。
- 目的：只保留最新楼层状态栏，避免旧楼层状态栏重复显示。

## 6. P1/P2 机制经验

### 6.1 P1 背包与装备目标

P1 背包不能出现 `玩家` 目标，也不能装备玩家。

P1 装备目标只允许五名 NPC：

- 白芷
- 苏芸
- 纪兰
- 沈月秋
- 柳素衣

P2 则保留 `己身` 语义，并写入底层 `道具.装备.玩家`。不要用 P1 修复破坏 P2 法器扣合。

### 6.2 P2 物品不是商城购买

P2 没有灵石消费逻辑。牝奴期物品不是商城商品，而是宗门调教、日课、执事库发付、惩戒或验器所得。前端读取玩家已装备状态，用它表现身体变化、羞名和日课后果。

红线：

- 不把执事库写成商城。
- 不把法器发付写成灵石购买。
- 不把 P1 普通道具混入 P2 法器匣。

### 6.3 日课闭环

日课要求玩家装备某类法器时：

- 如果 `道具.装备.玩家` 中存在对应法器，AI 应写身体和处境变化。
- 如果未装备，AI 可写执事斥责、公开点名、羞名入册或加重日课。
- AI 不得凭空替玩家装备法器。

### 6.4 烙名痕迹与身体留痕

二者不能重复：

- `身体留痕`：肉身、法器、牝印、改塑、淫纹等可见或可感的变化。
- `烙名痕迹`：宗门如何称呼、记录、传播和处置玩家，是社会身份与羞名层。

## 7. MVU 与世界书经验

### 7.1 JSONPatch 协议

当前协议使用 `add` 追加数组，禁止使用 `insert`。关键白名单包括：

- `add /牝奴/调教记录/-`
- `add /剧情/事件记录/-`
- `add /系统/风声列表/-`
- `add /系统/心音回响/-`

后端 sanitizer 要兼容外部解析器把 `add /数组/-` 转成内部 `insert 数组 '-' value` 的情况，但世界书和提示词仍要求 AI 写 `add`。

### 7.2 时辰兼容

`系统.时辰` 仍保持四段枚举：

- 晨时
- 午时
- 酉时
- 亥时

记录类字段需要容错旧楼层或 AI 写入的 `辰时/戌时` 等值，避免 ZodError 导致状态栏崩溃。`validate.ts` 应尽量归一化，schema 做防崩兜底。

### 7.3 前端与 AI 边界

前端可以：

- 读 MVU 变量。
- 写 `系统.待处理交互`。
- 做库存、扣费、装备、按钮反馈等即时 UI 更新。

前端不能：

- 直接替 AI 生成剧情结果。
- 直接调用 `generate()` 推进剧情。
- 伪造心音、惩戒、羞名或法器改造结果。

## 8. 文案与世界观经验

玩家主体感必须保留：

- 玩家仍清醒。
- 玩家会羞耻、抗拒、复盘和判断。
- P2 不是直接精神投降，而是身体和处境被规训。

NPC 修改后必须检查三件事：

1. 是否贴合攻略链或 P2 职责。
2. 是否破坏系统闭环。
3. 是否出现现代、出戏或越权行为。

成人向文风指南应区分日常和亲密场景尺度。发布宣传文案不写露骨细节，用“欲海、命数、宗门凝视、多结局”等钩子制造神秘感。

发布前至少完成：

- `pnpm vitest run "src/雌堕合欢宗"`
- `pnpm build:cloud`
- `node tavern_sync.mjs bundle 雌堕合欢宗`
- 如果头像为空，将 `dist/雌堕合欢宗.png` 改为 `dist/雌堕合欢宗.json`
- 检查 JSON：
  - 可 `JSON.parse`
  - 包含 `chara_card_v3`
  - 包含 `cdn.jsdelivr.net/gh/chen986347310-svg/tavern_helper_template@main/public/hehuan`
  - 不包含 `hehuan-cloud-assets`
  - 不包含 `localhost:5500`
- purge `cdn.jsdelivr.net` 五个运行文件。
- 用 CDN 检查入口页新版脚本。
- 导入酒馆后点击四个入口，确认可切换开场白。

## 10. 过期路线

以下路线已废弃，后续不要恢复：

- 本地 `5500` 端口发布。
- 私有仓库直接给 `testingcf` 读取。
- 独立 `hehuan-cloud-assets` 发布仓库。
- 角色卡 JSON 内容伪装为 `.png` 直接发布。
