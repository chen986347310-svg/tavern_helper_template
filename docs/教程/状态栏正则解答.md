# 项目构建流程

## src vs dist 的区别

| 目录 | 内容 | 能否直接运行 |
|------|------|-------------|
| `src/` | 源代码（Vue、TypeScript、SCSS） | ❌ 不能 |
| `dist/` | 编译后的产物（HTML、CSS、JS） | ✅ 可以 |

## 构建流程

```
src/雌堕合欢宗/
├── App.vue          ← Vue 组件（需要编译）
├── 界面/
│   ├── pages/*.vue  ← Vue 组件（需要编译）
│   └── styles/*.scss ← SCSS（需要编译）
└── index.ts         ← TypeScript（需要转译）
        ↓
    webpack 编译（npm run build 或 npm run watch）
        ↓
dist/雌堕合欢宗/
└── index.html       ← 可直接在浏览器运行
```

## 状态栏加载方式

酒馆通过正则替换 `<StatusPlaceHolderImpl/>` 为 HTML 代码来加载状态栏：

```html
<script>
$('body').load('http://localhost:5500/dist/%E9%9B%8C%E5%A0%95%E5%90%88%E6%AC%A2%E5%AE%97/index.html')
</script>
```

**为什么加载 dist：**
- dist/index.html 是 webpack 编译后的最终产物
- 包含所有 Vue 组件编译后的 JavaScript
- 包含所有 SCSS 编译后的 CSS
- 可直接在浏览器中运行，无需额外处理

**开发流程：**
1. 修改 `src/` 中的源代码
2. 运行 `npm run build` 或 `npm run watch` 编译
3. 生成 `dist/` 中的可运行文件
4. 酒馆通过正则替换加载 `dist/` 中的文件
